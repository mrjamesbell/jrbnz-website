Writing posts & pages

Posts and pages use Markdown. Common syntax:

# Heading 1Large heading
## Heading 2Section heading
**bold**bold
*italic*italic
[link](https://…)Hyperlink
![alt](https://…)Image
- itemBullet list
1. itemNumbered list
---Horizontal rule

Inserting images

Use the image button in the editor toolbar. You can upload a new image or pick one from the media library. Layout options: full-width, left float, right float, centred.

You can also paste a URL directly into the image block dialog.

Embedding YouTube

Paste a YouTube URL on its own line — it will be converted to a full embed automatically. Or use the toolbar button to set width and alignment.

https://www.youtube.com/watch?v=dQw4w9WgXcQ

Snippets

Snippets are reusable styled HTML blocks. Use the Insert snippet button in the editor toolbar to pick one. Current built-ins:

  • Stats bar — a row of large numbers with labels (e.g. years, shows, cities)
  • Two-column grid — side-by-side content blocks with headings and text
  • Timeline — a vertical timeline with year, title, company, and optional tag/award

Custom snippets can be created and managed in the Snippets view.

Site navigation

The site nav is built automatically from CMS pages with Include in site navigation checked. Order is: Now → Blog → Photos → everything else.

You can set a custom Nav link URL on a page to point the nav link somewhere else — useful for external sites.

Publishing

Hitting Publish on a post or page writes the HTML to R2 immediately — the live site updates within seconds. No deploy needed.

If you change navigation settings (adding/removing pages from the nav), use Rebuild site in the settings panel to regenerate all posts and pages with the updated nav.
