Help

Writing posts & pages

Posts and pages use Markdown. Common syntax:

# Heading 1Large heading
## Heading 2Section heading
### Heading 3Sub-heading
**bold**bold
*italic*italic
`code`Inline code
[link](https://…)Hyperlink
![alt](https://…)Inline image
- itemBullet list
1. itemNumbered list
---Horizontal rule
> textBlockquote

Quote hierarchy

There are three ways to quote text, each with a different visual weight.

Blockquote — the standard > syntax. In the cinematic theme these render as a gold left rule at 60% column width. Good for a cited passage or a key line set apart from prose. Use for: one or two sentences, incidental to the surrounding text.

> People often talk about carrying secrets to the grave.

Pull quote — a .pull-quote div wrapping a blockquote. Renders in the Baskerville display font, slightly wider than the reading column, with top and bottom rules. Use for: a sentence you want the reader to stop at, editorial emphasis, one defining idea per section.

<div class="pull-quote">
> The best productions feel inevitable in retrospect.
</div>

Quote interlude — a .quote-interlude div. Full-width dark section break with large Baskerville text. Use sparingly — once per long piece at most. Good for a structural pause between major sections, or a central idea that earns its own moment.

<div class="quote-interlude">
> You cannot separate the performance from the space it inhabits.
</div>

Inserting images

Use the image button in the editor toolbar to upload or pick from the media library. After choosing an image you will be asked to select a layout role and treatment.

Layout roles

WideBreaks slightly outside the reading column. Best for landscape, atmospheric, or location images.
BreakFull-width cinematic pause. Best for mood images that reset the reader's attention between sections.
SmallModest image within the reading column. Best for archival, documentary, or reference images.
PairTwo images side by side. Insert two consecutive images both set to Pair — the site wraps them automatically.

Treatment classes

MutedReduced saturation and contrast. Recommended default for most atmospheric photos.
MonoStrong monochrome. Use for archival, cinematic, stark, or memory-like images.
SoftLower contrast, slightly lifted. Use for reflective, quiet, nostalgic, or landscape images.
ColourMostly untreated. Use when colour matters — not as a default.

Common combinations: Wide + Muted for most unrelated-but-atmospheric photos. Break + Mono for a cinematic structural pause. Small + Colour for a documentary reference image. Pair + Soft for two related landscape or reflective images.

The cover image for a post is set in the post settings panel. Use a landscape image at least 1600×900px.

Type system

The cinematic theme uses three typefaces, each with a distinct role.

  • Baskerville (display) — used for hero titles, section headings, pull quotes, quote interludes, and the homepage featured title. Tight letter-spacing and a low line-height. This is the voice of the site.
  • Georgia (body) — used for all running prose: article paragraphs, blockquote text, decks, post excerpts. Generous line-height for comfortable reading.
  • System monospace — used for labels, kickers, metadata, nav items, and the Signal admin interface. Uppercase with wide letter-spacing for UI chrome.

You do not set these in post content — they are applied automatically by the theme based on context. If you use a .pull-quote or .quote-interlude wrapper, Baskerville is applied automatically to the quote inside.

Homepage

The homepage has four curated sections. Go to the Homepage view in the rail to edit them.

  • Featured article — a large hero at the top. Defaults to the most recent essay. You can pin a specific post and override its title, deck, image, and CTA label.
  • Three cards — links to the major site sections (Essays, Photographs, and a third destination). You can configure title, kicker, link, and style for each. Leave all three blank for the defaults.
  • Interlude — a full-width atmospheric moment, text-only or with a background image. Used to establish mood and editorial voice.
  • Archive section — a short list of curated older posts. Pin up to three by slug, or leave blank to auto-select recent essays.

Clicking Save & publish rebuilds the homepage immediately — no full site rebuild needed.

Embedding YouTube

Paste a YouTube URL on its own line — it will be converted to a full embed automatically. Or use the toolbar button to set width and alignment.

https://www.youtube.com/watch?v=dQw4w9WgXcQ

Code blocks

Wrap code in triple backticks. Optionally add a language name for syntax highlighting.

```javascript
const x = 1;
```

For short inline snippets, use single backticks: `like this`.

Snippets

Snippets are reusable styled HTML blocks. Use the Insert snippet button in the editor toolbar to pick one. Custom snippets can be created in the Snippets view.

Site navigation

The site nav is built automatically from CMS pages with Include in site navigation checked. You can set a custom Nav link URL on a page to point the nav link somewhere else — useful for external sites.

Publishing

Hitting Publish on a post or page writes the HTML to R2 immediately — the live site updates within seconds. No deploy needed.

If you change navigation settings (adding/removing pages from the nav), use Rebuild site in Settings to regenerate all posts and pages with the updated nav.

Editorial approach

Not every post needs a literal image. Use images that add something the text cannot — mood, texture, temporal context, visual contrast. An atmospheric photograph of a staircase can be more useful than a photograph of the person being written about.

Muted and mono treatments exist to keep images from competing with the prose. Use photo-colour sparingly — primarily when colour is genuinely important (a specific place, a person, an object that the article is directly about).

Pull quotes and quote interludes should be used once per piece, if at all. Their power comes from restraint. If every section has a pull quote, none of them work.
